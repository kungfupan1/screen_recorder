# 录屏王 项目技术总结文档

## 一、项目概述

录屏王（Screen Recorder King）是一款基于 Python 开发的 Windows 屏幕录制桌面应用程序。采用 PySide6 构建图形界面，支持全屏录制、区域录制、多显示器同时录制、麦克风采集、系统声音采集、音视频合并输出等功能。内置免费/付费分级体系，通过 Ed25519 签名实现离线许可证验证。

### 3.1 main.py — 程序入口

**职责**：初始化 Windows DPI 感知、设置环境变量、启动 Qt 应用。

#### 全局变量

| 变量名 | 类型 | 作用 |
|--------|------|------|
| APP_DIR | str | 应用程序所在目录的绝对路径 |

#### 函数

| 函数名 | 参数 | 返回值 | 作用 |
|--------|------|--------|------|
| main() | 无 | 无 | 创建 QApplication、设置全局字体、创建并显示主窗口、启动事件循环 |

#### 启动流程

1. 调用 `ctypes.windll.shcore.SetProcessDpiAwareness(2)` 启用 Per-Monitor DPI Aware V2
2. 设置 `QT_AUTO_SCREEN_SCALE_FACTOR=0` 和 `QT_ENABLE_HIGHDPI_SCALING=0`，禁用 Qt 内置缩放
3. 判断 `sys.frozen` 确定应用目录（支持 PyInstaller 打包）
4. 创建 `QApplication`，设置应用名称"录屏王"、版本号"1.0.0"
5. 使用 `UI_SCALE` 计算全局字体大小并设置
6. 创建 `MainWindow` 并显示

### 3.2 utils/config.py — 配置与缩放管理

**职责**：DPI 缩放计算、资源路径解析、默认配置。

#### 全局变量

| 变量名 | 类型 | 作用 |
|--------|------|------|
| UI_SCALE | float | DPI 缩放因子，`max(dpi / 96.0, 1.0)` |
| DEFAULT_CONFIG | dict | 默认配置 `{"fps": 30, "output_dir": "recordings", "format": "mp4"}` |

#### 函数

| 函数名 | 参数 | 返回值 | 作用 |
|--------|------|--------|------|
| get_app_dir() | 无 | str | 获取应用根目录（支持打包后路径） |
| get_resource_path(relative_path) | relative_path: str | str | 获取资源文件绝对路径（打包后取 `sys._MEIPASS`） |
| _get_dpi_scale() | 无 | float | 读取 Windows 系统 DPI，计算缩放因子 |
| sc(size) | size: int | int | DPI 缩放：`int(size * UI_SCALE)`，用于固定尺寸元素 |
| wsc(size, zoom) | size: int, zoom: float | int | DPI + 窗口缩放：`max(1, int(size * UI_SCALE * zoom))`，用于随窗口缩放的元素 |

#### 类：Config

| 属性/方法 | 说明 |
|-----------|------|
| fps | 帧率，默认 30 |
| output_dir | 输出目录，默认应用目录下的 `recordings` |

### 3.7 recorder/controller.py — 录制控制器

**职责**：管理屏幕录制和音频录制的完整生命周期。

#### 类：ScreenRecorder

单个屏幕的录制器，运行两个线程（捕获线程 + 写入线程），通过 FFmpeg 子进程管道编码。

**类变量**

| 变量名 | 类型 | 作用 |
|--------|------|------|
| _cached_encoder | str | 内存缓存的编码器名称 |

**实例属性**

| 属性名 | 类型 | 作用 |
|--------|------|------|
| width | int | 录制区域宽度（调整为偶数） |
| height | int | 录制区域高度（调整为偶数） |
| bounds | tuple | 录制区域 (x, y, width, height) |
| output_path | str | 输出视频文件路径 |
| fps | int | 帧率 |
| frame_count | int | 已录制帧数 |

**停止流程（异步）**

1. 设置所有 `_stop_event`（瞬间完成，不阻塞 UI）
2. 停止音频采集
3. 设置 `_saving = True`
4. 启动后台线程：并行等待所有 recorder 线程结束 + FFmpeg flush + 音频合并
5. 后台完成后设置 `_saving = False`

- 字节 10-13：过期时间戳（4 字节，大端序）
- 字节 14-77：Ed25519 签名（64 字节）
